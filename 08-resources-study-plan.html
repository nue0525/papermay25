Study Plan

8-Week Preparation Timeline

Weighted by domain importance. Spend more time on Domain 1 (27%) and hands-on building than on memorizing facts.

Weeks 1–2 · Domain 1 (27%)
Agentic Architecture & Orchestration
Build a complete agentic loop with stop_reason. Implement coordinator + 2 subagents. Practice parallel execution. Study anti-patterns. Build escalation logic with hooks. Complete Exercise 1 from exam guide.
Week 3 · Domain 2 (18%)
Tool Design & MCP Integration
Write differentiated tool descriptions. Implement structured error responses. Configure .mcp.json. Practice tool_choice: "auto" vs "any" vs forced. Study Grep vs Glob. Complete Exercise 2.
Weeks 4–5 · Domain 3 (20%)
Claude Code Configuration & Workflows
Set up full CLAUDE.md hierarchy. Create path-specific rules. Build skills with frontmatter. Practice plan mode vs direct. CI integration with -p flag. Study non-existent flags (CLAUDE_HEADLESS, --batch).
Week 6 · Domain 4 (20%)
Prompt Engineering & Structured Output
Build extraction pipeline with tool_use. Implement validation-retry. Write few-shot examples. Practice batch processing strategy. Design multi-pass review. Complete Exercise 3.
Week 7 · Domain 5 (15%)
Context Management & Reliability
Practice "case facts" extraction. Implement scratchpad files. Study escalation decision patterns. Design error propagation. Build human review workflows. Complete Exercise 4.
Week 8 · Final Review
Practice Exams & Weak Area Review
Complete official practice exam. Review all wrong answers against exam guide. Re-study weakest domain. Focus on scenario format — understand why wrong answers are wrong. Take the exam!
Study Technique from Exam Passers
Upload the 45-page official Exam Guide PDF to Claude and ask it to interview you conceptually about each domain. Don't do multiple choice — explain concepts in your own words. This is more effective than passive reading.
Pre-Exam Checklist

Final Readiness Check

Can you answer "yes" to every item? If not, study that area before exam day.

Domain 1: Agentic Architecture (27%)
  • Can you implement an agentic loop with stop_reason control from memory?
  • Can you explain 3 anti-patterns for loop termination?
  • Can you design a coordinator-subagent system with explicit context passing?
  • Do you know when to use hooks vs prompt instructions?
  • Can you explain PostToolUse and tool interception hook patterns?
  • Do you know when to use fork_session vs --resume vs fresh session?
Domain 2: Tool Design & MCP (18%)
  • Can you write differentiated tool descriptions that prevent misrouting?
  • Can you implement structured error responses with errorCategory and isRetryable?
  • Do you know tool_choice "auto" vs "any" vs forced — when to use each?
  • Can you configure .mcp.json with env var expansion?
  • Do you know Grep (content search) vs Glob (path match) distinction?
Domain 3: Claude Code Config (20%)
  • Can you draw the CLAUDE.md hierarchy (user/project/directory)?
  • Do you know .claude/commands/ (project) vs ~/.claude/commands/ (personal)?
  • Can you create .claude/rules/ with YAML frontmatter glob patterns?
  • Do you know context: fork, allowed-tools, argument-hint for skills?
  • Can you explain plan mode vs direct execution with examples?
  • Do you know -p flag for CI and that CLAUDE_HEADLESS doesn't exist?
Domain 4: Prompt Engineering (20%)
  • Can you design JSON schemas with nullable fields, enum+"other"+detail pattern?
  • Do you know tool_use eliminates syntax errors but NOT semantic errors?
  • Can you implement a validation-retry loop with error feedback?
  • Do you know Batch API constraints (24h, no latency SLA, no multi-turn)?
  • Can you explain why self-review is ineffective vs independent instances?
Domain 5: Context & Reliability (15%)
  • Can you explain "lost in the middle" effect and mitigation?
  • Do you know the 3 valid escalation triggers?
  • Do you know why sentiment-based escalation is unreliable?
  • Can you design structured error propagation across multi-agent systems?
  • Can you explain claim-source mappings for information provenance?
Master Cheat Sheet

Universal Decision Framework

The exam's recurring pattern: "What went wrong?" → Identify root cause → Choose proportionate first fix.

The CCA Decision Hierarchy
If the problem is...The answer is usually...NOT...
Agent skips required stepsProgrammatic hook (deterministic)Better prompt (probabilistic)
Agent calls wrong toolImprove tool descriptions (root cause)Few-shot examples (symptom treatment)
Research coverage gapsFix coordinator decompositionBlame downstream agents
Subagent missing contextExplicitly pass in promptAssume auto-inheritance
Output format inconsistentFew-shot examples (2-4)More detailed prose instructions
False positives too highExplicit categorical criteria"Be more conservative"
Need guaranteed JSONtool_use + JSON schema"Respond in JSON" prompt
Model fabricates missing dataNullable schema fieldsBetter prompt instructions
Same session reviews own codeIndependent review instance"Be extra critical" prompt
Blocking workflow needs savingsKeep real-time, batch only non-blockingBatch everything
Context fills with verbose dataTrim tool output via hooksLarger context window
Exact values lost over timePersistent "case facts" blockBetter summarization
Customer wants humanEscalate immediatelyResolve first, then offer
Team can't access toolsMove to project .mcp.jsonShare ~/.claude.json
CI job hangs-p flagCLAUDE_HEADLESS (doesn't exist)
Final Advice
The exam is proctored — no Claude, no docs, no tools. You need to KNOW this material, not look it up. Focus on understanding the "why" behind each answer, not memorizing answers. If you can explain the reasoning to someone else, you're ready.
