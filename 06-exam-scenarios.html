1
Customer Support Resolution Agent
Build a customer support agent using Claude Agent SDK. Handles returns, billing disputes, account issues. Access backend via MCP tools: get_customer, lookup_order, process_refund, escalate_to_human. Target: 80%+ first-contact resolution while knowing when to escalate.
D1: Agentic Architecture 27%D2: Tool Design 18%D5: Context & Reliability 15%

Topics You'll Be Tested On:

Prerequisite enforcement: Block process_refund until get_customer returns verified ID. Programmatic hooks, not prompt instructions.
Tool description quality: Why get_customer and lookup_order get confused when descriptions are minimal. Fix: expand descriptions with boundaries.
Escalation calibration: Agent escalates easy cases but handles complex ones. Fix: explicit criteria with few-shot examples.
Context preservation: Exact refund amounts lost to summarization. Fix: persistent "case facts" block.
Multi-concern decomposition: Customer has 3 issues in one message. Decompose, investigate each, synthesize unified response.
Structured handoffs: When escalating, compile: customer ID, root cause, amount, steps taken, recommended action.
Hook patterns: PostToolUse for date normalization. Tool interception for blocking refunds > $500.
2
Code Generation with Claude Code
Use Claude Code for code generation, refactoring, debugging, and documentation. Configure custom slash commands, CLAUDE.md, plan mode vs direct execution.
D3: Claude Code Config 20%D5: Context & Reliability 15%

Topics You'll Be Tested On:

CLAUDE.md hierarchy: User (~/.claude/CLAUDE.md) vs project (.claude/CLAUDE.md) vs directory. New team member missing config = user-level config not shared.
Slash command locations: .claude/commands/ (team) vs ~/.claude/commands/ (personal). .claude/config.json commands array DOES NOT EXIST.
Plan mode vs direct execution: 45-file monolith refactoring = plan mode. Single-file bug fix = direct execution.
Path-specific rules: .claude/rules/ with YAML frontmatter paths: ["**/*.test.tsx"] for cross-directory test conventions.
Skills with frontmatter: context: fork isolates verbose output. allowed-tools restricts destructive actions. argument-hint prompts for input.
Iterative refinement: Input/output examples > prose descriptions. Test-driven iteration. Interview pattern for unfamiliar domains.
3
Multi-Agent Research System
Coordinator delegates to search, analysis, synthesis, and report subagents. System researches topics, produces comprehensive cited reports.
D1: Agentic Architecture 27%D2: Tool Design 18%D5: Context & Reliability 15%

Topics You'll Be Tested On:

Narrow decomposition: "AI in creative industries" → coordinator only decomposes into visual arts. Root cause: coordinator, not downstream agents.
Context isolation: Subagents don't auto-inherit coordinator context. Must explicitly pass prior findings in prompt.
Parallel execution: Multiple Task calls in single coordinator response for parallel subagent spawning.
Scoped verify_fact: Give synthesis agent limited verification tool for 85% common case; complex verifications through coordinator.
Error propagation: Timeout returns structured context (failure type, partial results, alternatives), not generic "unavailable."
Information provenance: Claim-source mappings preserved through synthesis. Conflicting stats annotated, not arbitrarily selected.
Iterative refinement: Coordinator identifies coverage gaps → re-delegates with targeted queries → re-invokes synthesis.
4
Developer Productivity with Claude
Build tools to explore codebases, understand legacy systems, generate boilerplate. Uses built-in tools (Read, Write, Bash, Grep, Glob) + MCP servers.
D2: Tool Design 18%D3: Claude Code Config 20%D1: Agentic Architecture 27%

Topics You'll Be Tested On:

Grep vs Glob: Grep = search file CONTENTS (find function callers). Glob = find files by NAME pattern (**.test.tsx).
Incremental exploration: Grep → Read → trace flows. NOT reading all files upfront.
Edit vs Read+Write: Edit for unique text matches. Read+Write fallback when Edit fails on non-unique text.
MCP server config: .mcp.json (project) vs ~/.claude.json (personal). Env var expansion for secrets.
Scratchpad files: For extended sessions when agent starts referencing "typical patterns" instead of discovered specifics.
Subagent delegation: Spawn subagents for specific investigations while main agent coordinates.
5
Claude Code for CI/CD
Integrate Claude Code into CI/CD for automated code reviews, test generation, PR feedback. Design prompts for actionable feedback with minimal false positives.
D3: Claude Code Config 20%D4: Prompt Engineering 20%

Topics You'll Be Tested On:

-p flag: Non-interactive mode. CLAUDE_HEADLESS and --batch DO NOT EXIST.
--output-format json + --json-schema: Structured output for automated PR commenting.
Avoiding duplicate comments: Include prior findings in context, instruct to report only new/unaddressed issues.
False positive management: Explicit categorical criteria > "be conservative." Temporarily disable high-FP categories.
Self-review limitation: Same session reviews own code = ineffective. Use independent review instance.
Multi-pass review: Per-file local analysis + cross-file integration pass for large PRs (14+ files).
Batch processing: Batch overnight reports (50% savings). Real-time for blocking pre-merge checks (no latency SLA).
6
Structured Data Extraction
Build extraction system using Claude. Extract from unstructured documents, validate with JSON schemas, maintain high accuracy. Handle edge cases gracefully.
D4: Prompt Engineering 20%D5: Context & Reliability 15%

Topics You'll Be Tested On:

tool_use + JSON schema: Guaranteed syntax compliance. Eliminates JSON errors but NOT semantic errors.
Nullable fields: Prevent hallucination by making absent-data fields nullable instead of required.
Enum + "other" + detail: Extensible categorization with structured fallback.
tool_choice options: "auto" (may return text), "any" (must call tool), forced (must call specific tool).
Validation-retry loop: Send original doc + failed extraction + specific errors. Retries help for format errors, not missing data.
Few-shot for varied formats: Examples showing extraction from inline citations vs bibliographies, narrative vs structured tables.
Human review routing: Field-level confidence scores calibrated with labeled validation sets. Stratified sampling for error rate measurement.
97% accuracy trap: Aggregate accuracy may mask poor performance on specific document types or fields. Always segment before automating.
